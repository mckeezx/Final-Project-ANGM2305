# Title
Blackjack: Simplified

## Repository
https://github.com/mckeezx/Project-Proposal

## Description
The code will run a session of Vegas Strip Blackjack. The player starts with $100 and to win must reach triple their amount ($300).
The project is relevant to media and digital arts as it's a (player) interactive system that uses UI (information layout) feedback loops, and randomness through a card game. 


## Features
- Blackjack System
	- The core game loop:
        deal cards out
        player can choose to hit/stand (loop)
        dealer turn (follows rules set)
        compare results
        give or take money
    - Uses loops, conditionals, lists, functions

- Vegas Strip Mechanics
	- Double Down (player can double their bet in exchange for one more card, then must stand) 
        Uses conditional 
    
    - Splitting (player can split their hand up to 3 times)
        Use conditional for each hand, compare each hand to the dealer
    
- Tutorial System
	- Create a start up screen that explains keybinds and perhaps the rules 
        Just makes use of print()

- Stylized UI
    - takes place in the Terminal
    - spacing, alignment, color, ASCII, clearing screen, print()

## Challenges
- Splitting Hands (and the logic behind it in python)
- Stylized UI/text (and how to make it look pretty)
- Rules and anything to know about Vegas Strip Blackjack (how dealer works, how players work)

## Outcomes
Ideal Outcome:
- A fully finished game that plays a session of Vegas Strip Blackjack. Players can split hands, double their hands, hit, and stand. The dealer will work. There will be no rigging, just relying on math and standards of Blackjack. The UI also works well with the game. 

Minimal Viable Outcome:
- The game can run a session of regular Blackjack. Player can hit and stand, and the dealer can work. UI serves just to convey information. Money system also works and player can win the session or lose depending on if they triple or lose their money.
